# Security Policy

## Supported versions

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Report security issues through one of these channels:

1. **GitHub Security Advisories (preferred):**  
   [Create a private vulnerability report](https://github.com/laithtak/Quorum/security/advisories/new)

2. **Repository maintainer:** Open a draft security advisory or contact the maintainer through GitHub.

We aim to acknowledge reports within 72 hours and will work with you on a fix and coordinated disclosure timeline.

## Scope

This policy covers the Quorum repository, including:

- `council-ai` Python package and CLI
- Web UI (`web/`)
- GitHub Actions workflows
- Docker images published from this repo

Out of scope: third-party LLM provider APIs, misconfiguration of API keys in user environments, and issues in dependencies already tracked upstream.

## Safe harbor

We appreciate good-faith security research. Do not access data that is not yours, disrupt services, or violate applicable laws.
